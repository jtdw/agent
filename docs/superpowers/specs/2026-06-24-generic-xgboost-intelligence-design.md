# 通用 XGBoost 智能建模设计

## 目标

将普通 XGBoost 请求默认路由到通用建模工作流，使其可用于农作物产量等回归、森林覆盖等分类，以及表格、矢量属性、点位-栅格采样和栅格堆栈数据。上传后的本地数据画像自动识别任务类型、空间/时间条件与验证策略。智谱仅接收脱敏画像，为特征工程、建模方案和结果解释提出结构化建议；本地规则和验证器始终决定是否采纳与执行。

## 非目标

- 不移除既有土壤水分专项流程；它保留为用户明确选择时的专项工具。
- 不将原始记录、原始坐标、上传文件、模型产物或受保护路径发送给外部模型。
- 不允许外部模型直接调用工具、训练模型或生成未经本地计算验证的指标。
- 不改动已有 API 的必填参数或响应字段；只增加可选输入和 diagnostics。

## 架构与数据流

1. `asset_profiler` 在上传或上下文构建阶段产出建模画像：数据类型、字段类型、数值/类别/日期/坐标字段、缺失率、唯一值数、样本数、几何类型和 CRS。
2. 本地 `modeling_profile` 从该画像提出候选目标、特征、回归/分类判断，以及验证策略。候选特征排除 ID、名称、几何、疑似泄漏字段和全空字段。
3. 当智谱配置启用且用户允许时，后端仅发送脱敏 `modeling_profile`。返回值必须符合受限 JSON schema，内容限于字段候选、排除理由、任务类型、验证策略、调参重点与解释建议。
4. `plan_validator` 检查智谱建议中的字段存在性、类型、样本量、敏感信息与数据泄漏风险。未通过时回退到本地建议并记录原因。
5. `task_planner` 对普通 XGBoost 请求默认选择 `generic_xgboost_workflow`，并将已验证的 `task_type`、验证策略及可选调参/SHAP 参数传入。
6. `generic_xgboost_workflow` 在本地执行预处理、切分、可选调参、训练、预测、真实指标、特征重要性和可选 SHAP。结果、参数、切分依据和限制写入 artifact 与 diagnostics。

## 路由与兼容性

- `XGBoost`、`XGB`、`预测模型`等普通请求选择 `generic_xgboost_workflow`。
- 明确提及土壤水分专项或专项工作流名称时，仍可选择既有 `train_xgboost_fusion_model` 或 STM 工作流。
- 用户提供 `task_type`、`split_method`、目标列或特征列时优先使用，但必须经过本地验证。
- 原有工具参数保持可用；新增的调优、解释和策略字段均为可选，并具备安全默认值。

## 自动任务与验证策略

任务类型按照以下顺序确定：用户明确值、本地目标字段类型与唯一值数、经验证的智谱建议。数值连续目标使用回归；文本或有限类别使用分类；不确定时要求用户确认，不猜测。

验证策略按可用数据选择：

- 有有效日期字段：时间留后切分，测试数据严格晚于训练数据。
- 有点几何或有效坐标：空间网格分块切分，减少空间自相关导致的高估。
- 同时具备时间和空间：先按时间留后，再在训练期内按空间块划分调参/验证；结果明确标注为时空留后验证。
- 有明确分组字段：分组切分。
- 缺少上述信息：随机切分，并将低泛化可信度写入 diagnostics。

若样本数、类别分布或空间块数量不足以支持选定策略，工作流必须返回结构化错误或安全回退，并记录原因；不能声称完成未实际执行的验证。

## 调参与解释

- 默认关闭高成本调参，用户请求“自动调参”或配置启用时启动受预算限制的随机搜索。
- 回归以 RMSE 最小化为主；分类优先使用 F1，二分类可使用 AUC。评分必须由实际验证结果产生。
- 参数搜索次数、交叉验证折数、随机种子和耗时上限写入 diagnostics。数据超过上限时采用可复现抽样并提示限制。
- SHAP 为可选能力：依赖可用时生成全局特征贡献与有限样本解释；不可用或计算超限时，保留模型内置特征重要性并返回结构化状态，不使训练失败。

## 智谱边界与失败处理

智谱请求体只允许包含字段名、推断类型、缺失率、唯一值数、范围或聚合统计、样本数量、空间/时间布尔元数据和本地候选策略。禁止包含行级数据、完整经纬度、文件名路径、凭据、artifact 内容或隐私字段值。

智谱超时、返回非 JSON、schema 不匹配、字段建议无效或服务未配置时，必须静默回退为本地画像策略，并在 diagnostics 记录 `advisor_status`。这不应阻断本地建模。

## 测试与验收

- 作物产量表：识别为回归，默认路由通用工作流，生成真实回归指标。
- 森林覆盖表：识别为分类，保留类别目标并生成分类指标。
- 空间点数据：选择空间分块；含时间字段的点数据：选择时空留后策略。
- 无空间/时间字段表：回退随机切分并写入可信度警告。
- 智谱建议有效时可被采纳；建议无效、超时或未配置时可安全回退。
- 断言智谱请求载荷不包含行级值、坐标值、路径、密钥或敏感字段。
- 调参与 SHAP 的依赖缺失、预算超限和正常输出均有覆盖。

## 修改范围

预计涉及 `core/asset_profiler.py`、新增的建模画像/顾问模块、`core/task_slots.py`、`core/task_planner.py`、`core/plan_validator.py`、`core/ml/generic_xgboost.py`、现有智谱客户端接口以及对应测试。不会重新设计前端整体布局；如需展示，仅在现有结果卡片增加模型策略与顾问状态。
