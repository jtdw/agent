# 地理空间 XGBoost 建模工作流

本文档是 Planner 参考，不得代替 Plan Validator、Data Asset Profiler 或 XGBoost 工具的确定性检查。模型指标、特征重要性和产物必须来自真实 ToolResult。

## 输入角色

建模任务通常需要目标变量、特征变量和训练样本。目标可以来自站点表、矢量属性或已验证的目标栅格。特征可以来自表字段、矢量属性、栅格采样结果或栅格堆栈。站点-栅格建模还需要真实坐标或点几何，以及必要时的时间字段。

## 空间和时间匹配

站点与栅格匹配必须基于真实坐标和 CRS。若存在时间维度，应按用户指定或明确确认的时间窗口匹配。时间不匹配时不能猜测，应提出具体策略问题，例如“按同日、月平均还是最近日期匹配？”

## 训练与验证

训练前应检查目标列、特征列、缺失率、有效样本量和训练/验证切分。空间交叉验证需要空间分组字段或坐标；没有空间信息时不能声称完成空间验证。可报告的指标必须来自真实结果，例如 RMSE、MAE、R2、NSE、残差摘要和特征重要性。

随机划分只适合样本近似独立的场景。地理样本存在空间自相关时，随机划分可能高估泛化能力；若用户关心空间泛化，应优先考虑空间分组、空间块或留区验证。时间序列或多期站点数据还应避免时间泄漏：训练集不能使用验证期之后的信息，也不能用未来栅格或未来统计量作为特征。

训练测试独立性必须来自真实切分和输入元数据。若同一站点、同一像元或同一时间窗口同时泄漏到训练和测试，指标不能解释为独立验证结果。随机种子应记录在 diagnostics 或 outputs 中，便于复现；如果 ToolResult 没有返回随机种子，最终回复不能自行编造。

缺失值和异常值处理应由工具或已验证计划明确执行。缺失值过多、目标列异常、特征单位混杂或单位不明时，应在 warnings 或 errors 中说明限制。特征单位和含义应来自字段元数据、用户说明或真实数据字典，不能由 LLM 猜测。

有效样本量不足时不应强行训练。样本量阈值以当前工具和 Validator 实际检查为准；知识库只能提示风险，不能替代校验。

## 常见失败处理

- 样本不足：停止训练或要求更多样本。
- 字段缺失：指出缺少的目标列或特征列。
- CRS 不一致：仅在计划内允许重投影且输入完整时修复，否则澄清或失败。
- 时间不匹配：提出具体匹配策略问题，不能自动选择。
- 空间泄漏或时间泄漏风险：说明验证指标可能偏乐观，并建议用户确认切分策略。
- 缺失值、异常值或单位不明：先报告真实诊断，再请求用户确认处理策略。
- 训练失败：展示真实错误摘要和可执行 next_actions。
