# 课程作业项目介绍

## 项目名称

GIS 智能体工作台

## 项目简介

本项目是一个面向地理数据处理的 GIS 智能体系统。用户可以上传地理数据，并通过中文自然语言描述分析需求。系统会识别数据类型、字段、坐标系和空间范围，选择合适的 GIS 工具或稳定工作流，完成空间分析、地图预览、结果解释和产物下载。

与普通聊天机器人不同，本项目强调“智能体 + GIS 工具链 + 可复现工作流”。智能体不只是生成文本，而是围绕真实地理数据执行工具，生成可查看、可下载、可追踪的结果。

## 技术组成

- 前端：React、TypeScript、Vite，负责上传、对话、地图预览和结果展示。
- 后端：FastAPI，负责 API、会话、文件、工具调用和 artifact 管理。
- 智能体运行时：负责组织上下文、工具卡片、规划结果和执行边界。
- GIS 工具链：支持矢量、栅格、表格、制图、建模等处理任务。
- 工程验证：包含 pytest、前端构建、E2E smoke、staging observation gate 和 CI 分层。

## 核心能力

- 上传并识别地理数据。
- 通过中文对话触发 GIS 分析任务。
- 执行矢量裁剪、栅格统计、表格经纬度转点、地图制图等工作流。
- 生成地图预览、结构化结果、artifact 和下载产物。
- 支持土壤湿度 XGBoost 建模和 GeoConformal Prediction 不确定性输出的扩展能力。
- 通过 CI 和 staging gate 保证关键流程可复现。

## 项目特点

- 面向 GIS 真实任务，而不是泛聊天。
- 工具执行有参数校验、路径安全、会话隔离和 artifact 注册。
- 支持中文文件名、中文字段和中文提示词。
- 具备工程化收尾证据：代码合并到 main、CI 通过、关键 smoke gate 可复现。

## 当前状态

项目已完成本地运行、核心功能、测试和 CI 收尾。真实远端 staging 尚未执行，因为缺少真实环境的 host、admin token 获取方式、服务重启方式和部署配置核验路径。该部分已作为后续运营接入项记录在项目文档中。
